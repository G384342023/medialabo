「Webコンテンツ作成」の成果

本テーマについて

先行する情報工学の専門科目の基礎知識を応用して実践的な能力を修得することを目的とする。 到達目標は、手順書を読み、実験を行い、レポートやプレゼンテーションにより報告するという実験の一連の流れを実践できることと、HTML等を用いたWebページ作成ができることである。

プロジェクトの課題について

Webページの概要

  • 検索キー内から一つを選択する。
  • 検索ボタンをクリックする。
  • 選択されたジャンルに応じた結果が出力される。
  • コンソール内でAJAX通信の処理が行われる

Webページの要素

  • プルダウンメニュー
  • 検索ボタン
  • 結果表示エリア

Webページのロジック

print関数が以下の処理を行う。

  1. ユーザーが選択したジャンルの情報を取得
  2. 選択されたジャンルに基づいてAPIのURLを構築
  3. APIからデータを取得し、showResult関数を呼び出す

showResult関数が以下の処理を行う。

  1. APIからのレスポンスデータを解析
  2. 検索結果の件数を表示
  3. 各店舗の情報を整形してHTML要素として追加

showError関数とfinish関数が以下の処理を行う。

  1. showError: エラーをコンソールに表示
  2. finish: 検索終了メッセージをコンソールに表示

プロジェクト課題のページ: project.html

数当てゲーム

ゲームの概要

  • 1から10までのランダムな数字が選ばれる。
  • プレイヤーは3回まで予想できる。
  • 各予想後にフィードバックが表示される。

数当てゲームの出力パターン

  1. 正解の場合：
    • 正解です．おめでとう!
  2. 不正解で、試行回数が1回目または2回目の場合：
    • 予想が小さすぎる場合：まちがい．答えはもっと大きいですよ
    • 予想が大きすぎる場合：まちがい．答えはもっと小さいですよ
  3. 不正解で、試行回数が3回目の場合：
    • まちがい．残念でした答えは 000 です．
  4. 試行回数が3回を超えた場合：
    • 答えは 000 でした．すでにゲームは終わっています
    • （注：000は実際の答えの数字に置き換わります）

ゲームの要素

  • 数字入力ボックス
  • 回答送信ボタン
  • 結果表示エリア

ゲームのロジック

hantei関数が以下の処理を行う。

  1. プレイヤーの入力と正解を比較
  2. 適切なフィードバックメッセージを生成
  3. 残り試行回数の管理

HTMLの要素がリアルタイムで更新され、プレイヤーに直感的なゲーム体験ができる。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第1回 WWW と HTML

Web通信におけるプロトコルは約束事であり、一般的にはhttpsが利用される。 サーバはWebページの配布元を指し、パスはサーバ上のファイル配置場所を示す。 通信はブラウザからのページ要求で発生し、クライアントサーバシステムに基づいて行われる。 ブラウザとサーバの間の通信はHTTPプロトコルに従う。WebページはHTML、CSS、JavaScript、画像・音声・動画の部品から構成され、ブラウザはこれらを組み合わせて表示する。 またHTMLの基本操作を行った。 HTMLとはWEBサイトを制作する際に、コンピュータへ構成指示を出し、表示したい文章などの情報を形作るマークアップ言語の一つである。

演習の成果: 演習1-4: HTMLの様々な要素

第2回 CSS による Web ページの装飾

CSS（Cascading Style Sheets）は、HTML文書の見た目を制御するためのスタイルシート言語だ。 レイアウト、色、フォントなどのデザイン要素を指定し、ウェブページの外観を一貫して整えるために使用される。 CSSを用いたwebページの制作をした。

演習の成果: 演習2-3: Webブラウザ説明ページの説明

第3回 JavaScript の基本

JavaScriptは、Webページ上で複雑な機能を実現するためのプログラミング言語だ。 ページを開いた直後やボタンを押したときに実行され、ページの自動更新やアニメーションなど動きのあるページを作成する。 ブラウザAPIとして、DOM API、Geolocation API、CanvasとWebGL、WebRTCなどがあり、これらによりHTML要素の操作や現在位置の取得、2D・3Dグラフィックスの描画、リアルタイム通信が可能である。 また、Twitter APIやGoogle Map APIなどのサードパーティAPIも利用できる。 JavaScriptはインタプリタ言語であり、プログラムは1行ずつ読み取り実行している。 このため、コンパイルの手間がなく、手軽に実行とデバッグが可能だ。 JavaScriptは主にクライアントサイドコードとしてブラウザ上で動作するが、サーバサイドコードと区別することが重要である。

演習の成果: 演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

DOMは、Webコンテンツを木構造で表現するデータモデルだ。 HTML文書の要素、属性、テキストがそれぞれノードとして表され、親子関係で連結される。 最上位にはdocumentノードがあり、すべてのノードの起点とな理うる。 JavaScriptを使用してDOMを操作することで、Webページの動的な変更が可能になる。 プログラムからDOMのノードをオブジェクトとして取得し、内容の読み取りや変更、新しいノードの追加などができる。 これにより、ページのコンテンツや外観をプログラムで動的に制御できるようになり、インタラクティブなWebページの作成が可能となる。

演習の成果: 演習4-2~4-5: DOMプログラミング

第5回 イベント処理

JavaScript のイベント処理は、ウェブアプリケーションの対話性を高める重要な機能である。 主にaddEventListenerメソッドを使用し、要素にイベントリスナーを追加する。 これにより、クリックやキー入力などのユーザーアクションに対して特定の処理を実行できる。 イベントオブジェクトを活用することで、発生したイベントの詳細情報にアクセスできる。例えば、クリックされた要素や入力されたキーコードなどを取得できる。 イベントの伝播制御も重要な概念である。キャプチャリングフェーズとバブリングフェーズを理解し、必要に応じてstopPropagationメソッドを使用することで、イベントの伝播を制御できる。 非同期処理との組み合わせにより、ユーザー体験を損なうことなく複雑な処理を実行することも可能である。

演習の成果: 練習5-3: イベントを選択する
