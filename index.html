第1回 WWW と HTML

Web通信におけるプロトコルは約束事であり、一般的にはhttpsが利用される。 サーバはWebページの配布元を指し、パスはサーバ上のファイル配置場所を示す。 通信はブラウザからのページ要求で発生し、クライアントサーバシステムに基づいて行われる。 ブラウザとサーバの間の通信はHTTPプロトコルに従う。WebページはHTML、CSS、JavaScript、画像・音声・動画の部品から構成され、ブラウザはこれらを組み合わせて表示する。 またHTMLの基本操作を行った。 HTMLとはWEBサイトを制作する際に、コンピュータへ構成指示を出し、表示したい文章などの情報を形作るマークアップ言語の一つである。

演習の成果: 演習1-4: HTMLの様々な要素

第2回 CSS による Web ページの装飾

CSSを用いたwebページの制作をした。 CSS（Cascading Style Sheets）は、HTML文書の見た目を制御するためのスタイルシート言語だ。 レイアウト、色、フォントなどのデザイン要素を指定し、ウェブページの外観を一貫して整えるために使用される。

演習の成果: 演習2-3: Webブラウザ説明ページの説明

第3回 JavaScript の基本

JavaScriptは、Webページ上で複雑な機能を実現するためのプログラミング言語だ。 ページを開いた直後やボタンを押したときに実行され、ページの自動更新やアニメーションなど動きのあるページを作成する。 ブラウザAPIとして、DOM API、Geolocation API、CanvasとWebGL、WebRTCなどがあり、これらによりHTML要素の操作や現在位置の取得、2D・3Dグラフィックスの描画、リアルタイム通信が可能である。 また、Twitter APIやGoogle Map APIなどのサードパーティAPIも利用できる。 JavaScriptはインタプリタ言語であり、プログラムは1行ずつ読み取り実行している。 このため、コンパイルの手間がなく、手軽に実行とデバッグが可能だ。 JavaScriptは主にクライアントサイドコードとしてブラウザ上で動作しますが、サーバサイドコードと区別することが重要である。

演習の成果: 演習3-6: オブジェクトの参照
